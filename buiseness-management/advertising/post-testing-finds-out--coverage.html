EXPLANATIONS BELOW

Concept note-1: -How to Calculate Test Coverage. Calculating test coverage is actually fairly easy. You can simply take the number of lines that are covered by a test (any kind of test, across your whole testing strategy) and divide by the total number of lines in your application.

Concept note-2: -The test coverage report provides information about parts of the software where test coverage is being implemented. Essentially, it provides information about the tests executed on an application or website.

Concept note-3: -Function coverage is a metric measuring the functions invoked during software testing. The number of functions executed by a test suite is divided by the total number of functions in the software under testing to calculate this metric.

Concept note-4: -Test coverage helps you unearth areas of a program that have not been covered by a set of test cases. It helps make your program more robust and error-free.
